# Spec

## Tese

`nested panel` so deve existir quando funcionar como continuacao imediata da voz do hero.

## Regras

1. o painel aninhado precisa prolongar a leitura do hero, nao repetir a mesma mensagem em outra caixa
2. o painel aninhado nao pode competir com o hero por protagonismo
3. o painel aninhado deve anteceder a execucao, nao substituir a execucao
4. se a tela ja possui focus lane ou command lane forte abaixo do hero, o `nested panel` entra so se remover duplicacao, nunca se somar camadas

## Checklist pratico de revisao por tela

1. o hero continua sendo a voz principal da abertura?
2. o `nested panel` prolonga a mesma historia em vez de reiniciar a introducao?
3. existe alguma toolbar, faixa ou intro logo abaixo repetindo a mesma funcao?
4. o primeiro bloco de execucao continua claramente abaixo da camada de orientacao?
5. no mobile, hero e `nested panel` continuam parecendo um conjunto e nao dois cards competindo?
6. se o `nested panel` sumisse, a tela perderia continuidade real ou so simetria visual?
