# Context

## Locked Decisions

1. This mountain should harden the live class grid contract, not redesign the whole class-grid product.
2. Hardcoded edit links must move toward named routes or canonical route-building strategy.
3. Inline behavior and presentational debt should leave markup when a clear owner already exists.
4. Weekly and daily class-grid surfaces should feel like the same product, not like two local dialects.
5. The current editing and deletion workflow must remain operationally clear for roles that can manage classes.

## Why This Mountain Exists

The class grid is one of the most visible operational surfaces left with mixed interaction ownership:

1. hardcoded edit links
2. inline style in interactive cards
3. clickable behavior expressed directly in markup
4. duplicated interaction patterns across daily and weekly views

In simple terms:

1. the class board is already on the wall
2. but some magnets, labels, and shortcuts are still improvised
