# Spec

## Problem

The conversion drawer is no longer structurally messy, but it remains dormant and disconnected from the live onboarding runtime.

## Requirements

### DCFD-01

The decision must be based on the actual live intake flow, not on dormant code alone.

### DCFD-02

The package must end with a clear recommendation: reactivate or retire.

### DCFD-03

If reactivation is recommended, the next-step work must be explicit.

### DCFD-04

If retirement is recommended, the safe removal path must be explicit.
