# C.O.R.D.A. Plan

## Context

Reception is the next governed expansion surface after dashboard and finance.

## Objective

Bring premium cohesion to reception without sacrificing speed, clarity, or counter-level readability.

## Risks

1. importing too much glow or atmosphere can slow the screen mentally
2. making reception too editorial can weaken action-first behavior
3. copying dashboard language directly would recreate visual debt

## Direction

Use the premium grammar in a quieter register:

1. stronger surfaces
2. cleaner command framing
3. better shell cohesion
4. restrained visual energy

## Actions

1. map which reception zones can absorb the grammar
2. define the quiet translation rules
3. implement the reception pass
4. validate that the counter still feels fast
