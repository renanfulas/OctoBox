# Design

## Translation principle

Reception should inherit premium grammar as a quiet counter surface, not as a glowing stage.

## What can travel

1. stronger surface layering
2. cleaner shell cohesion
3. more mature border/shadow discipline
4. more intentional hero framing

## What must stay restrained

1. glow intensity
2. background atmosphere
3. emotional all-clear styling
4. anything that makes all blocks compete equally for attention

## Core question

Can reception feel more premium while still behaving like:

1. a fast operational counter
2. a human-facing service surface
3. a next-action-first workspace

instead of:

1. a mini-dashboard
2. an editorial hero page
3. a surface that looks expensive but thinks slowly

## Translation result

The implemented reception pass follows this hierarchy:

1. shell + hero
   - richest treatment, but still quieter than dashboard and finance
2. intake + payment
   - command-weighted surfaces with premium depth
3. class-grid + shortcuts
   - support-weighted surfaces with calmer contrast and atmosphere

This keeps the reception counter readable in the first seconds while still moving it into the same premium family as the rest of the product.
