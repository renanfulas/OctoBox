# Spec

## Goal

Expand the premium OctoBox theme into reception in a quieter operational register that increases cohesion and perceived product maturity without weakening action-first usage.

## In scope

1. `templates/operations/reception.html`
2. reception shell, hero, and command surfaces
3. reception support boards where visual depth can improve cohesion

## Out of scope

1. dashboard-only premium theatrics
2. finance-specific hero behavior
3. payload, routing, or business-logic changes unless strictly necessary

## Success criteria

1. reception feels more premium and more unified with dashboard/finance
2. reception remains fast to scan in the first seconds
3. command and next action stay more prominent than support surfaces
4. the result respects Front Display Wall and does not become scenery
