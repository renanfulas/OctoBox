# Context

## Locked Decisions

1. The canonical theme is already the official visual authority.
2. Legacy hosts such as `glass-panel-elite` must not remain as sovereign structure in shared components.
3. Shared components deserve SDD because they can affect multiple surfaces at once.

## Why This Mountain Exists

The short post-governance review found that the shared metric card component still carries a legacy premium shell:

1. `card glass-panel-elite metric-card-elite`

Because this is a shared component, the risk is multiplicative.

In simple words:

- this is not a crooked picture frame on one wall
- this is the mold that prints frames for several rooms

## Boundaries

This feature should not become:

1. a dashboard redesign
2. a rewrite of every metric presentation in the app
3. a generic search-and-replace without visual judgment

The scope is the shared metric card contract and the CSS that truly owns it.
