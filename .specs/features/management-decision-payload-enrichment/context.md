# Context

## Why this exists

The recent cleanup cycle made the front more honest:

1. less inline behavior
2. fewer hardcoded routes
3. clearer reading order
4. more explicit flow between sections

That changed the bottleneck.

The next ceiling is not visual cleanliness.

It is payload intelligence.

Right now, several strategic surfaces still depend on the front to infer urgency from correct but flat data.

## Problem framing

The product already knows:

1. overdue pressure
2. intake backlog
3. schedule occupancy
4. student and payment movement

But the snapshots do not always express:

1. why this is the first thing to read
2. what action dominates now
3. what can wait
4. what the user gains by acting on this item first

In simple terms:

the house now has good signs, but the control tower still speaks too softly.

## Non-goals

This feature is not for:

1. redesigning the UI from scratch
2. reopening broad CSS cleanup
3. introducing new visual systems
4. bloating payloads with decorative prose

## Success shape

After this pass, management surfaces should receive data that already carries:

1. urgency
2. reason
3. recommended next move
4. clearer distinction between live pressure and contextual depth
