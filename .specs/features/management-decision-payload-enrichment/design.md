# Design

## Core thesis

This is a control-tower pass.

Not a paint pass.

The UI should receive management data that already says:

1. this is the fire
2. this is the smoke
3. this is just weather around the building

## Design principles

### 1. Dominant pressure first

Each strategic surface should expose one dominant operational pressure before secondary context.

### 2. Reason beats decoration

If a payload field does not help the user decide sooner, it should not exist.

### 3. Next move must stay attached

Priority without action is just anxiety.

Enriched signals should stay close to:

1. where to go next
2. why now
3. what outcome that move protects

### 4. Local intelligence, not duplicated prose

Snapshots should carry the reason.

Templates should still own wording cadence and facade rhythm.

## Target surfaces

### Dashboard

Goal:

1. make the headline cards feel less static
2. clarify which metric is just informative and which one is pressing

### Finance Center

Goal:

1. sharpen queue vs movement vs portfolio priority
2. expose why a queue deserves to open first when pressure exists

### Operations

Goal:

1. keep owner and adjacent management flows grounded in canonical next actions
2. reduce any remaining payload ambiguity around where to go first

## Allowed moves

1. add compact semantic fields
2. reorder based on business pressure
3. attach next-action rationale
4. promote live pressure above contextual depth

## Forbidden moves

1. giant schema explosions
2. prose-heavy payloads
3. UI reinvention disguised as payload work
4. performance-blind enrichment
