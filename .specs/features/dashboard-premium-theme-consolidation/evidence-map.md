1. premium atmosphere can increase magnetism
2. operational truth can remain intact
3. dashboard-local success still needs grammar before reuse

## T1 pilot inventory by visual role

### Layer 1. Exportable shell grammar candidates

Anchors:

1. `static/css/design-system/dashboard.css`
2. `static/css/design-system/topbar.css`

Observed roles:

1. deeper navy shell temperature
2. layered support surfaces instead of flatter dark blocks
3. shell-to-topbar integration via scoped variables
4. quieter premium borders, inset highlights, and shadows

Why this matters:

These feel like product grammar candidates, not dashboard gimmicks.

### Layer 2. Semi-exportable surface treatment

Anchors:

1. `static/css/design-system/components/dashboard/side.css`

Observed roles:

1. richer support-card depth
2. premium but quieter rail surfaces
3. side-card polish that still stays subordinate to the main command strip

Why this matters:

This looks reusable in quieter variants, but probably not as literal copy-paste.

### Layer 3. Dashboard-local alert theater

Anchors:

1. `static/css/design-system/components/dashboard/glance/glance_cards.css`
2. `static/css/design-system/components/dashboard/glance/glance_neon.css`

Observed roles:

1. stronger glance-card mass and premium status-card presence
2. actionable glow tied to urgency semantics
3. all-clear state as a dashboard-specific emotional release
4. alert pulse behavior whose exact intensity is tightly tied to dashboard reading

Why this matters:

The principles are good, but the exact treatment feels much closer to dashboard-local identity than to generic product grammar.

## T2 classification result

### Export bucket

1. premium navy shell temperature
2. layered support-surface construction
3. scoped topbar integration grammar
4. premium border and inset-light discipline
5. semantic neon rule: severity first, glow second

### Adapt bucket

1. support-rail card depth
2. richer card atmosphere for secondary surfaces
3. hero richness where command authority exists
4. quieter premium variants for finance or reception

### Keep-local bucket

1. exact glance pulse amplitude
2. exact actionable glow values in the topline
3. all-clear emotional treatment
4. dashboard-specific stage balance between main strip and side rail
