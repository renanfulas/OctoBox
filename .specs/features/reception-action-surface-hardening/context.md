# Context

## Locked Decisions

1. Operational surfaces deserve more caution than cosmetic surfaces.
2. Inline style and inline behavior inside a live payment action card create real maintenance risk.
3. WhatsApp follow-up is part of the workflow value, so hardening must preserve it.

## Why This Mountain Exists

The reception payment card still mixes:

1. markup
2. inline style
3. inline behavior
4. WhatsApp action copy
5. privilege-driven action gating

This is a live operational surface, not a placeholder.

In simple words:

- this is not wallpaper
- this is the front desk where money and communication pass through together

## Boundaries

This feature should not become:

1. a reception module redesign
2. a full finance communication refactor
3. a rewrite of all WhatsApp flows in the system

The scope is the payment action surface and its immediate ownership.
