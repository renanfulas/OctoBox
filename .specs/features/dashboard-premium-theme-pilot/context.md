# Context

## Why this pilot exists

The OctoBox front-end is now far more synchronized and governable.

That changes the next question from:

- "how do we clean the front?"

to:

- "how do we give the product a stronger, more magnetic face without betraying operational truth?"

The repository `Refactoruiuxdesign` offers a visual language that is highly compatible with OctoBox:

1. navy dark mode
2. glassmorphism
3. controlled neon accents
4. premium cards with strong hierarchy
5. cleaner shell atmosphere

But it should not be transplanted wholesale.

In simple terms:

- it is a good donor of facial structure
- not a full skeleton replacement

## Runtime facts confirmed before opening this package

1. the dashboard is the strongest pilot surface for premium visual translation
2. `dashboard.css`, `topbar.css`, and glance card files already carry the main dashboard atmosphere
3. the current dashboard already respects decision-first semantics, so the pilot must amplify that instead of replacing it

## Boundaries

This package does:

1. translate the compatible visual language from the reference repository into OctoBox dashboard surfaces
2. preserve decision-first hierarchy
3. keep visual intensity disciplined and operationally honest

This package does not:

1. import React, Tailwind, or Shadcn code
2. redesign the whole product at once
3. spread the pilot to finance, reception, or student form before the dashboard proves itself

## Non-goals

1. no global theme migration in one pass
2. no direct code transplantation from the reference repository
3. no decorative premium layer that competes with urgency and next action
