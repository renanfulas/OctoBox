# CORDA Plan

## North

Translate the strongest visual language from `Refactoruiuxdesign` into the OctoBox dashboard so the product feels more premium, magnetic, and unified while remaining decision-first.

## Objective

Make the dashboard feel more like the polished front of a premium operational product and less like a technically correct control board with uneven atmosphere.

## Risks

### Risk 1. Decorative drift

The dashboard could become more beautiful but less truthful.

### Risk 2. Theme transplant debt

Borrowing too much from the reference repo could create a second visual system inside OctoBox.

### Risk 3. Intensity overload

Glass, neon, and gradients can quickly become noise if they are not localized and role-aware.

## Direction

1. import language, not code
2. strengthen the dashboard shell, topbar, and glance surfaces first
3. keep urgency, risk, and next action visibly dominant
4. treat premium atmosphere as disciplined lighting, not stage fog

## Actions

1. map the translation between reference tokens and OctoBox tokens
2. define what to import, adapt, and avoid
3. implement the pilot only on the dashboard
4. validate whether the dashboard becomes both more desirable and more decisive
