# Context

There is a live local change in `static/css/design-system/dashboard.css` that centers the advisor hero content more aggressively.

This is not random debris.

It is a composition hypothesis:

1. a more centered advisor hero may feel more editorial
2. a narrower main block may strengthen the dashboard's front-facing identity
3. the action row may feel more curated if it aligns to the same center mass

But this can also go wrong:

1. the hero may lose command energy
2. the screen may start feeling like a landing page instead of an operational front

This package exists to answer that question cleanly.
