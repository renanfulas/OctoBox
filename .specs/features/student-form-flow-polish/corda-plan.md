# C.O.R.D.A. Plan

## Context

The student form is a high-density product surface where many actions converge.

## Objective

Improve the student form so that users can understand where to start, how to recover, and when to move into finance without hesitation.

## Risks

1. overexplaining can make the screen slower instead of clearer
2. adding guidance in the wrong layer can create visual debt
3. changing sequence without respecting the real workflow can break trust

## Direction

Polish the flow in a way that:

1. shortens the mental path
2. clarifies the next right move
3. keeps the screen feeling operational, not tutorial-heavy

## Actions

1. map the live friction points in the student form
2. define the minimum sequence improvements with maximum clarity gain
3. implement the flow polish
4. validate that the form feels easier, not heavier
