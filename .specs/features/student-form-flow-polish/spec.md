# Spec

## Goal

Refine the student form so that the registration-to-finance journey is clearer, faster, and easier to recover when the user hits friction.

## In scope

1. `templates/catalog/student-form.html`
2. `catalog/presentation/student_form_page.py`
3. related student-form UI fragments directly involved in sequence or recovery
4. small supporting CSS/template changes required for flow clarity

## Out of scope

1. major visual-theme expansion
2. large backend business-rule changes
3. unrelated finance center or operations screens

## Success criteria

1. the first action is obvious
2. the finance transition feels earned and timely
3. recovery guidance feels useful without becoming noisy
4. the student form becomes easier to complete under real operational pressure
