# Specification

## Problem

Finance already has good structural clarity, but it still lacks the premium shell coherence that the dashboard pilot proved possible.

Without a guided expansion, future premium styling risks becoming:

1. inconsistent
2. too dramatic for a transactional surface
3. detached from the governance rules we just established

## Goal

Translate the premium theme grammar into finance by improving:

1. shell continuity
2. command-surface presence
3. support-card depth
4. premium coherence between hero, intro, boards, and support rails

## Non-goals

1. cloning dashboard glance behavior into finance
2. changing finance payload semantics
3. redesigning reception or student form in the same pass

## Success criteria

This expansion succeeds when:

1. finance feels more premium and more unified
2. queue, movements, and portfolio remain easy to parse under pressure
3. the new layer feels quieter than dashboard, not flatter than dashboard
