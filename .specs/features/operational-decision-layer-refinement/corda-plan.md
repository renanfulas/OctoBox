# C.O.R.D.A. Plan

## Context

The front has left the heavy sanitation phase and entered a product-polish phase.

The biggest strategic gain now is decision support:

1. what the user sees first
2. what feels urgent
3. what is context
4. what action should happen next

## Objective

Refine the operational decision layer across the most important live workspaces so each page communicates a clean decision sequence before depth and secondary context.

## Risks

1. over-editing and turning a prioritization pass into a redesign
2. adding explanatory blocks that increase noise instead of reducing it
3. making surfaces more explicit in wording but not actually clearer in hierarchy
4. touching too many files and losing the surgical nature of the pass

## Direction

Follow the Front Display Wall rule:

1. one clear reading entry point
2. one visible next action
3. context after command
4. depth after pressure

We should treat these pages like a guided corridor, not a room full of equally loud signs.

## Actions

### Wave 1

Map current decision flow on:

1. reception
2. student form
3. finance
4. advisor dashboard

### Wave 2

Refine local hierarchy where the order is still implicit.

### Wave 3

Validate whether each surface now answers, quickly:

1. what matters now
2. what is blocked or pressured
3. what happens next
