# Specification

## Problem Statement

OctoBox now looks much more coherent, but some of its most important workspaces still rely on the user's intuition to discover reading order.

The product should not only be clean.
It should also guide judgment.

## Users

1. reception operators handling arrival, payment, and class orientation
2. managers and operators using the student form during live service
3. finance operators reading pressure, queue, and portfolio
4. advisors reading follow-up, risk, and next move

## Requirements

### R1. Decision-first reading

Each targeted surface must reveal its primary operational sequence before deeper supporting content.

### R2. Command and context separation

Command surfaces must not visually compete with support surfaces.

### R3. Minimal structural intervention

The pass should refine hierarchy with the smallest safe footprint:

1. intro rails
2. sequence maps
3. stronger section ordering
4. local CSS adjustments

### R4. No visual dialect regression

Any new support block must be born from canonical hosts and semantic local classes.

## Success Criteria

The pass is successful when a teammate can open each target page and answer within a few seconds:

1. what is first
2. what is second
3. what is support
4. what action should happen now
