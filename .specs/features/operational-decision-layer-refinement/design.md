# Design

## Core thesis

This is not a redesign pass.

It is a decision-layer pass.

The page should behave like a good airport:

1. first you see where to go
2. then you see what line matters
3. then you see details

Not every sign should shout at once.

## Surface strategy

### Reception

Goal:

1. keep the workspace short
2. reinforce the three-pass counter rhythm
3. ensure intake, payment, and class orientation read as a sequence

### Student Form

Goal:

1. make identity and finance read in the right order
2. prevent the page from feeling like two equal blocks competing at once

### Finance

Goal:

1. clarify that cash pulse comes before queue pressure
2. clarify that queue pressure comes before portfolio depth
3. make filters feel like support, not the opening scene

### Advisor Dashboard

Goal:

1. preserve the current cleaned structure
2. only intervene if the advisory narrative still competes with action priority

## Allowed moves

1. short sequence intros
2. explicit numbering where helpful
3. section lead-ins
4. spacing and grouping adjustments
5. semantic local classes

## Forbidden moves

1. large visual reinvention
2. new parallel card systems
3. presenter rewrites unless absolutely required
4. adding copy that slows reading instead of clarifying it
