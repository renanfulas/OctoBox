# POR QUE EXISTE:
# - Centraliza pacotes usados em CI/local para rodar a suite de testes,
#   medir cobertura, gerar fakes/factories, mockar HTTP e relogio, e
#   detectar testes order-dependentes.
# - Mantem versoes pinadas (== em vez de >=) para que CI seja
#   reproduzivel — qualquer mudanca de pacote vira commit explicito.

# 2. cobertura (pytest-cov, coverage)

#
# E2E (Playwright/Selenium) NAO esta incluido aqui — quando o plano de
# qualidade de testes definir essa frente, adicionar pytest-playwright
# em uma segunda iteracao.
#
# PONTOS CRITICOS:
# - Versoes devem ser compativeis com Django==6.0.5 e Python 3.12.
# - Manter sincronizado com os pacotes que os workflows CI instalam.
